# DPLL 算法详解 (Davis-Putnam-Logemann-Loveland)

DPLL 是解决布尔可满足性问题（SAT）的经典完备算法，也是现代更高级的 CDCL 求解器的基础。它的本质是一个**带有剪枝机制的深度优先搜索（DFS）**，通过递归地给变量赋值并化简 CNF 公式来寻找解。

## 1. 核心流程 (The DPLL Procedure)
DPLL 算法的伪代码可以概括为以下四个主要步骤的循环与递归：

- **function** DPLL(CNF公式 $\Phi$, 当前赋值 $A$):
  - // 1. 基础情况 (Base Cases)
  - **if** $\Phi$ 为空 (没有任何子句): **return** SAT, $A$  // 所有子句都被满足了
  - **if** $\Phi$ 包含空子句 (即发生冲突): **return** UNSAT   // 存在永远无法满足的子句
  - // 2. 布尔约束传播 (BCP - Boolean Constraint Propagation)
  - **while** $\Phi$ 包含单文字子句 (Unit Clause) $L$:
    - 将 $L$ 赋值为 True 加入 $A$
    - 根据 $L$ 的值化简 $\Phi$
    - **if** $\Phi$ 产生空子句: **return** UNSAT
  - // 3. 纯文字消除 (Pure Literal Elimination)
  - **while** $\Phi$ 包含纯文字 $P$:
    - 将 $P$ 赋值为 True 加入 $A$
    - 删除所有包含 $P$ 的子句
  - // 4. 分支决策 (Branching / Decide)
  - 从 $\Phi$ 中选择一个未赋值的变量 $X$
  - // 递归尝试 $X = True$
  - **if** DPLL($\Phi \land \{X\}$, $A \cup \{X = True\}$) == SAT: 
    - **return** SAT
  - // 如果失败，回溯并尝试 $X = False$
  - **return** DPLL($\Phi \land \{\neg X\}$, $A \cup \{X = False\}$)

## 2. 三大核心机制拆解

### 2.1 布尔约束传播 (BCP) / 单文字规则 (Unit Rule)
* **定义**：如果一个子句中**只有一个未赋值的文字**，且其他文字都已经计算为 `False`，那么这个未赋值的文字**必须**被赋值为 `True`（否则该子句为假，整个 CNF 也会为假）。
* **作用**：BCP 是 SAT 求解器的“引擎”，能像多米诺骨牌一样产生连锁反应。一个单文字的赋值可能会在其他子句中产生新的单文字，从而极大地减少需要搜索的空间。
* **化简操作**：假设通过单文字规则，文字 $L$ 被赋值为 True：
  1. **删除**所有包含文字 $L$ 的子句（因为这些子句已经被满足了）。
  2. **剔除**其他子句中出现的文字 $\neg L$（因为 $\neg L$ 是 False，在 OR 运算中不起作用）。

### 2.2 纯文字规则 (Pure Literal Rule)
* **定义**：如果一个变量在整个 CNF 公式中，只以**一种极性**出现（即要么全是正文字 $X$，要么全是负文字 $\neg X$），那么它被称为纯文字。
* **作用**：我们可以直接把这个纯文字赋值为 `True`。这样做可以满足所有包含它的子句，且不会对其他子句产生任何负面影响。
* *注：现代高级 CDCL 求解器为了追求速度，由于维护纯文字状态的开销较大，通常会**放弃**纯文字规则，而把所有优化精力放在 BCP 的加速上（如 Two-Watched Literals 技术）。*

### 2.3 分支决策 (Branching)
* **定义**：当 BCP 和纯文字规则都无法再推导出新变量的值时，算法只能“猜”。即从剩下的变量中挑一个，假设它是 True 或 False，然后继续往下走。
* **回溯 (Backtracking)**：如果猜的这条路走不通（导致某个子句变成了空子句，即所有文字都是 False），算法就会退回来，尝试这个变量的另一个取值。
* **启发式策略 (Heuristics)**：怎么挑变量“猜”也是一门学问。优秀的启发式算法（如 VSIDS）能优先挑出最容易引发冲突的变量，尽早剪枝，从而让求解速度呈指数级提升。

## 3. 从 DPLL 到 CDCL
DPLL 的主要缺陷在于**“健忘”**。当它在某条分支上遇到冲突并回溯后，它会把产生冲突的原因抛之脑后。如果换了一条分支又遇到相似的环境，它还会重蹈覆辙。
**CDCL (冲突驱动子句学习)** 则在 DPLL 的基础上加入了“学习机制”。每次冲突发生时，CDCL 会分析冲突图，总结出一个**学习子句 (Learned Clause)** 并加入到公式中。这相当于求解器在说：“我已经知道 $X=True$ 且 $Y=False$ 会死路一条了，我要加一个约束 $(\neg X \lor Y)$ 提醒自己以后别再这么干！”。
