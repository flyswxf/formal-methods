# 命题逻辑与合取范式 (CNF)

在 SAT (布尔可满足性问题) 求解中，几乎所有的现代求解器（如 Minisat, CDCL 架构求解器）都要求输入的问题是以**合取范式 (CNF)** 的形式给出的。为了理解并实现 SAT 求解器，必须完全掌握以下四个核心基本概念。

## 1. 变量 (Variable)
* **定义**：命题逻辑中最基本的单元，也称为**布尔变量 (Boolean Variable)** 或命题变元。
* **取值**：每个变量只能取两个值之一：`True` (真, 1) 或 `False` (假, 0)。
* **符号表示**：通常用英文字母或带下标的字母表示，如 $P, Q, R$ 或 $x_1, x_2, x_3$。
* **在 SAT 求解器中的意义**：求解器的最终目的，就是为所有的变量找到一组赋值（或者证明找不到），使得整个逻辑公式为真。

## 2. 文字 (Literal)
* **定义**：一个**变量 (Variable)** 或者是它的**否定 (Negation)**。
* **分类**：
  * **正文字 (Positive Literal)**：变量本身。例如：$x_1$。当 $x_1$ 被赋值为 True 时，该文字为 True。
  * **负文字 (Negative Literal)**：变量的否定。例如：$\neg x_1$ (读作 NOT $x_1$)。当 $x_1$ 被赋值为 False 时，该文字才为 True。
* **在 DIMACS 格式中的表示**：
  正文字用正整数表示（如 `1` 代表 $x_1$），负文字用负整数表示（如 `-1` 代表 $\neg x_1$）。

## 3. 子句 (Clause)
* **定义**：由若干个**文字 (Literal)** 通过逻辑**或 (OR, $\lor$)** 运算符连接而成的表达式。也叫“析取子句”。
* **形式**：$C = (l_1 \lor l_2 \lor \dots \lor l_k)$，其中 $l_i$ 是文字。
* **真值条件**：
  * **满足 (Satisfied)**：只要子句中**至少有一个文字**为 True，整个子句就为 True。
  * **冲突/不满足 (Falsified/Conflict)**：只有当子句中**所有的文字**都被赋值为 False 时，整个子句才为 False。
* **例子**：$(x_1 \lor \neg x_2 \lor x_3)$。要让这个子句为真，可以令 $x_1 = \text{True}$，或者 $x_2 = \text{False}$，或者 $x_3 = \text{True}$。

## 4. 合取范式 (Conjunctive Normal Form, CNF)
* **定义**：由若干个**子句 (Clause)** 通过逻辑**与 (AND, $\land$)** 运算符连接而成的逻辑公式。简单来说，CNF 就是“子句的合取（AND）”。
* **形式**：$F = C_1 \land C_2 \land \dots \land C_m$，其中每个 $C_j$ 都是一个子句。
* **真值条件**：
  * **满足**：要使整个 CNF 公式为 True，**必须所有的子句都同时为 True**。
  * **不满足**：只要有**任何一个子句**为 False（即发生了冲突），整个 CNF 公式就立刻为 False。
* **例子**：
  $$ (x_1 \lor \neg x_2) \land (\neg x_1 \lor x_3 \lor x_4) \land (\neg x_3) $$
  这个 CNF 公式由 3 个子句组成。要满足它，必须同时满足：
  1. $x_1$ 为真 或 $x_2$ 为假。
  2. $x_1$ 为假 或 $x_3$ 为真 或 $x_4$ 为真。
  3. $x_3$ 为假（这意味着 $x_3$ 必须强行被赋值为 False，这在 SAT 求解中称为**单文字规则 (Unit Clause)**）。

## 💡 为什么 SAT 求解器偏爱 CNF？
任何形式的命题逻辑公式，都可以通过算法（如 Tseitin 变换）在线性时间内转换为等价的 CNF 形式。
CNF 具有极其扁平、统一的结构（外层全是 AND，内层全是 OR）。这种高度统一的数据结构极大地简化了 SAT 求解器中 **BCP (布尔约束传播)**、**冲突分析** 和 **子句学习** 的实现代码，使其能够达到极高的运行效率。
